using ibeo::common::sdk::lanes::CarriageWay = typedef CarriageWayIn6972

A CarriageWay represents one surface of a road and has a unique identifier.

The identifier is a combination of the CarriageWayType of the road and a number (e.g. A for CarriageWayType motorway and 1 represents A1).

In addition each CarriageWay holds a list of segments. Within one segment, the number of lanes is constant. If there are preceding and following segments, these segments are linked against each other. It is therefore possible to store multiple linked lists of segments within on CarriageWay (e.g. for different driving directions or if there are gaps between segments).

A CarriageWay is always the highest representation of a road. The segmentation is as following:

CarriageWay → CarriageWaySegment → Lane → LaneSegment

The connection is handled using smart pointers. Therefore it is not possible to create an instance of this class, but calling create() will return a shared pointer to a new CarriageWay.

using ibeo::common::sdk::lanes::LaneSegment = typedef LaneSegmentIn6972

A LaneSegment represents a Lane segment within a parent Lane. Each LaneSegment has a unique id within the parent Lane

A Lane Segment ends if the approximation error through straight bounding lines reaches a certain value, or if a CarriageWaySegment ends.

The segmentation of a whole CarriageWay is as following:

CarriageWay → CarriageWaySegment → Lane → LaneSegment

The connection is handled using smart pointers. Therefore it is not possible to create an instance of this class, but calling create will return a shared pointer to a new LaneSegment.
